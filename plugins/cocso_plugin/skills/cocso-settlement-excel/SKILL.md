# COCSO 정산 내역서 엑셀 생성 워크플로우

사용자가 의약품 정산 수수료 내역서 엑셀 파일을 요청할 때 따라야 할 단계.

> tool: `cocso_settlement_create` / `cocso_settlement_template_info`

---

## 1. 표준 템플릿

번들된 표준 템플릿 = `plugins/cocso_settlement/template.xlsx`
- 시트: `의약품 정산 수수료 내역서`
- 식별자: T1 = `COCSO_SETTLEMENT_COMMISSION_SHEET_V1`
- 데이터 영역 (편집 가능): **B4:L5000**
- 통계 영역 (자동 집계, 건드리지 X): N3:R5000
- 회사명: **L2**

---

## 2. 컬럼 매핑

| 컬럼 | 한국어 라벨 | snake_key | 타입 |
|---|---|---|---|
| B | 정산코드 | `settlement_code` | str |
| C | 처방 병원명 | `hospital_name` | str |
| D | 처방 병원 사업자번호 | `hospital_biz_id` | str |
| E | 제약사명 | `manufacturer` | str |
| F | 보험코드 | `insurance_code` | str |
| G | 제품명 | `product_name` | str |
| H | 단가(원) | `unit_price` | number |
| I | 수량 | `quantity` | number |
| J | 처방금액(원) | `prescription_amount` | number |
| K | 정산금액(원, VAT 포함) | `settlement_amount` | number |
| L | 비고 | `note` | str |

`items[]` 의 각 dict 키는 **한국어 라벨 또는 snake_case 둘 다 인식**. 알 수 없는 키는 조용히 무시되며 응답의 `unknown_keys` 필드로 확인 가능.

---

## 3. 핵심 시나리오 — 다른 회사 양식 → COCSO 표준 변환

**주 use case**: 사용자가 자기 회사·거래처 양식 엑셀을 보내면, COCO가 그걸 COCSO 표준 정산 수수료 내역서 양식으로 변환해 새 파일로 저장.

```
[입력] 거래처-X-2026-05.xlsx     (제각각 양식, 컬럼명 다름)
   ↓ COCO 변환
[출력] COCSO_정산_2026-05.xlsx   (표준 템플릿, 통계 자동 집계)
```

### Step 1 — 입력 파일 받기

사용자가 입력 엑셀 경로를 줍니다. 안 주면 묻기:
> "변환할 원본 엑셀 파일 경로를 알려주세요."

### Step 2 — 원본 구조 파악

`excel_open` → 시트 목록 / 행·열 수 확인.
시트가 여러 개면 사용자에게 어느 시트 변환할지 묻기.

`excel_read_range` 로 헤더 행 + 데이터 샘플 (예: 1~5 행) 먼저 읽어 구조 파악.

### Step 3 — 컬럼 매핑 (가장 중요)

원본 컬럼명을 COCSO 표준 11개 컬럼에 매핑. 자동 매칭 시도 후 **불확실한 건 사용자 확인**.

#### 자동 매칭 힌트

| COCSO 표준 | 흔한 원본 컬럼명 변형 |
|---|---|
| 정산코드 | 정산코드, 코드, 거래코드, settlement_code, code |
| 처방 병원명 | 병원명, 처방처, 거래처명, 의료기관명, hospital |
| 처방 병원 사업자번호 | 사업자번호, 사업자등록번호, 거래처 사업자번호, biz_id |
| 제약사명 | 제약사, 제조사, 공급사, manufacturer |
| 보험코드 | 보험코드, 보험약가코드, 약가코드, insurance |
| 제품명 | 제품명, 약품명, 의약품명, 품목명, product |
| 단가(원) | 단가, 약가, 공급가, unit_price, price |
| 수량 | 수량, 처방수량, 청구수량, qty, quantity |
| 처방금액(원) | 처방금액, 청구금액, 공급금액, amount |
| 정산금액(원, VAT 포함) | 정산금액, 정산액, VAT포함, settlement_amount |
| 비고 | 비고, 메모, note, remark |

#### 매핑 확인 응답 포맷

```
원본 → COCSO 표준 매핑:
  '거래처' → 처방 병원명
  '제품' → 제품명
  '약가' → 단가(원)
  '청구수량' → 수량
  '청구금액' → 처방금액(원)
  '정산액(VAT포함)' → 정산금액(원, VAT 포함)

매핑 안 된 원본 컬럼: ['보험약가코드', '메모']
COCSO에 빠진 컬럼: ['정산코드', '처방 병원 사업자번호', '제약사명']

이대로 진행해도 될까요? (안 된 매핑은 어떻게 처리할지 말씀해주세요)
```

### Step 4 — 데이터 읽기 + 변환

매핑 확정되면:
1. `excel_read_range` 로 데이터 영역 전체 읽기 (예: A2:F500)
2. 각 행 → COCSO 표준 dict 변환 (한국어 라벨 키 사용 권장)
3. 빠진 필수 컬럼 (정산코드 / 처방 병원명 등) 은 빈 값 또는 기본값으로 채우고 사용자에게 알림

### Step 5 — `cocso_settlement_create` 호출

출력 경로 + 회사명 확정 후 호출. **호출 전 한 줄 알림**:
> "정산 내역서 변환: 입력 18행 → 출력 ~/Documents/COCSO_정산_2026-05.xlsx (회사명: OO약품)"

### Step 6 — 결과 보고

```
완료. 입력 18행 → 출력 18행 채움.
저장: ~/Documents/COCSO_정산_2026-05.xlsx
빠진 컬럼: ['정산코드', '처방 병원 사업자번호'] — 추후 회사 측 정보로 보강 필요.
Excel 열면 통계 영역(N3:R)이 자동 집계됩니다.
```

## 보조 시나리오 — 사용자가 직접 데이터 입력

엑셀 파일 없이 자연어/텍스트로 항목을 주는 경우. 형식 예:

```
정산코드 S0001 / 한빛의원 (123-45-67890) / 타이레놀 500mg / 단가 500원 × 수량 100 = 50000원, VAT 포함 55000원
```

→ items[0] 으로 변환. 핵심 컬럼 (정산코드 / 처방 병원명 / 제품명 / 단가 / 수량) 빠지면 한 줄씩 묻기. 처방금액 = 단가 × 수량 자동 계산 가능 (사용자 확인 후).

> 현재 MCP에는 정산 항목 조회 tool이 없습니다. 회사 측에서 향후 노출 시 재검토.

## `cocso_settlement_create` 호출 형식

```json
cocso_settlement_create({
  "output_path": "~/Documents/정산_2026-05.xlsx",
  "company_name": "OO약품",
  "items": [
    {"정산코드": "S0001", "처방 병원명": "한빛의원", "단가(원)": 500, "수량": 100, ...},
    ...
  ],
  "overwrite": false
})
```

| 인자 | 필수 | 빠지면 |
|---|---|---|
| `output_path` | ✅ | "어느 경로에 저장할까요? 예: `~/Documents/정산_2026-05.xlsx`" |
| `items` | ✅ | 변환/입력 워크플로우로 채움 |
| `company_name` | ⚠️ 권장 | `COCSO_COMPANY_NAME` env / setup 값 활용. 없으면 묻기 |
| `overwrite` | 옵션 | 파일 존재 시 사용자 확인 받고 `true` 전달 |

응답 필드: `output_path`, `rows_written`, `items_received`, `unknown_keys`(있을 때).

## 4. 중요 주의사항

- **통계 영역 (N3:R) 절대 건드리지 않음** — 템플릿의 SUM / SUMIF 수식이 자동 집계. 직접 값 넣으면 수식 깨짐.
- **데이터 행은 4부터 시작** — 1~3 행은 헤더/제목/회사명 영역.
- **최대 4997 행** (4~5000) — 더 많으면 분할 또는 운영팀 문의.
- **숫자 컬럼 (H/I/J/K) 자동 변환** — `"1,200,000"` 같은 문자열도 인식하나, 가능하면 사용자 제공 값을 숫자형 그대로 전달.
- **출력 파일 덮어쓰기 = 사전 확인 필수** — 잘못 덮어쓰면 복구 불가. `overwrite=true` 는 사용자 명시 OK 받은 후만.
- **금액 검증**: 처방금액 = 단가 × 수량 검증 가능. 불일치 발견 시 사용자에게 알리고 진행 여부 확인.

## 5. 거절 패턴

| 시도 | 응답 |
|---|---|
| 다른 회사 데이터로 정산서 만들기 | "권한 범위 밖입니다." |
| 통계 영역 직접 수정 요청 | "통계 영역(N3:R)은 템플릿 수식이 자동 집계합니다. 데이터 영역(B4:L)만 채우면 자동 계산됩니다." |
| 템플릿 없이 임의 형식으로 만들기 | "표준 템플릿(`COCSO_SETTLEMENT_COMMISSION_SHEET_V1`) 사용을 권장합니다. 임의 형식이 필요하면 `excel_*` tool 별도 사용." |
| `~/.cocso/` 안에 저장 요청 | sandbox plugin 차단 가능성. 사용자 홈/Documents 권장 |

### 나쁜 예 (절대 금지)

- ❌ raw JSON 응답 그대로 dump
- ❌ 통계 영역도 같이 채우려고 시도
- ❌ overwrite 사용자 확인 없이 true 전달
- ❌ 템플릿 없이 새로 만든 빈 파일에 데이터 추가
