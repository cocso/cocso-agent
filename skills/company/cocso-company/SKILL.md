# COCSO 회사 소개

## 한 문장

COCSO(코쏘)는 **의약품 유통·정산·계약을 한 곳에서 관리하는 B2B 플랫폼** 입니다.

---

## 누가 씁니까

| 사용자 | 어떤 일에 |
|---|---|
| **제약사** | 유통·영업 채널 관리 |
| **CSO** (Contract Sales Organization, 영업대행) | 딜러 관리, 거래처 정산, 수수료 운영 |
| **정산자** (상위 위임자) | 하위 위임 관계 관리, 정산 정책 운영 |
| **병·의원 / 약국** | 거래처로서 계약·정산 참여 |

→ COCO가 직접 응대하는 대상은 주로 **CSO** 와 **정산자** (= COCSO를 도입한 **비즈니스사**).

---

## 무엇을 디지털화합니까

| 기존 방식 | COCSO |
|---|---|
| 엑셀 + 수기 정산 | 자동 수수료 계산 (일·월 단위) |
| 종이/팩스 위임 계약 | 전자서명 + PDF 자동 생성 |
| 메신저로 재위임 통지 | 카카오 알림톡 / 이메일 자동 발송 |
| 분리된 의약품 마스터 | 외부 의약품 정보 자동 동기화 |
| 시스템 간 수기 연동 | Webhook / MCP 양방향 통신 |

---

## 플랫폼 구조

COCSO는 세 가지 백엔드로 구성됩니다.

### 1. 서버 (Server) — 핵심 비즈니스 API

사용자 역할별 7개 영역:

| 영역 | 대상 | 용도 |
|---|---|---|
| **Admin** | 내부 운영팀 | 사용자·회사·의약품·수수료·단가·프로모션 통합 관리 |
| **Client** | CSO (비즈니스사) | **딜러·거래처·정산 코드·수수료율** — COCO 주 사용 영역 |
| **Contract** | CSO + 정산자 | **위임·재위임 계약, 템플릿, 게스트 서명** — COCO 주 사용 영역 |
| **Service** | 외부 시스템 | 자격 조회, 제안, MCP API 키 관리 |
| **Settler** | 정산자 | 위임 관계 그래프, 의약품 수수료, 재위임 통지 |
| **Webhook** | 외부 시스템 | 이벤트 수신 |
| **MCP Webhook** | 외부 MCP 도구 | AI 에이전트 연동 창구 |

> COCO가 비즈니스사 사용자를 도울 때 직접 호출하는 영역은 **Client** 와 **Contract** 두 곳입니다. 다른 영역은 권한 밖.

### 2. 워커 (Worker) — 데이터·연산 자동화

정기 스케줄로 백그라운드 처리:

- **Commission** — 기간 단위 수수료 정산 계산
- **Daily Commission** — 일 단위 수수료 집계
- **Drug Info** — 외부 의약품 정보 동기화
- **Stock** — 재고·거래 데이터 처리
- **Webhook** — 외부 시스템으로 나가는 알림 전송

비즈니스사는 결과를 **조회·해석**할 뿐 직접 조작하지 않습니다.

### 3. 계약 워커 (Contract Worker) — 계약 자동화 전담

- 계약서 **PDF 자동 생성**
- **전자서명** 연동 + 진행 상태 관리
- **카카오 알림톡 / 이메일** 발송
- **재위임 통지** 발송·접수
- 고객 서명·문의 **지원 채널**
- 스케줄 기반 **일괄 처리**

---

## COCO가 비즈니스사를 돕는 영역

COCO는 COCSO의 일부로 작동하며, 비즈니스사가 다음 일을 자연어로 처리할 수 있도록 보조합니다:

1. **영업 운영 (Client)** — 딜러·거래처·정산 코드·수수료율 조회·수정 안내
2. **계약 처리 (Contract)** — 위임·재위임 계약 템플릿, 전자서명 요청, 진행 상태 확인, 재위임 통지 발송
3. **수수료 확인** — 자동 정산 결과(일·월) 조회와 해석
4. **알림 흐름 점검** — 카카오 알림톡 / 이메일 발송 상태 확인
5. **외부 연동** — Webhook·MCP 호출 안내

## COCSO MCP

COCSO는 두 종류의 MCP 서버를 제공합니다. COCO는 환경 변수만 채워두면 두 서버에 자동으로 연결됩니다.

### 두 MCP 비교

| 항목 | Client MCP | Service MCP |
|---|---|---|
| 서버 이름 | `cocso-client` | `cocso-service` |
| 대상 영역 | **Client 서버** (영업·정산 운영) | **Service 서버** (외부 노출 API) |
| 주 용도 | 비즈니스사 일상 영업 — 딜러·거래처·수수료·정산 그룹 | 자격(eligibility) 조회, 제안(proposal), 재위임 통지, MCP API 키 관리 |
| 누가 사용 | 비즈니스사 사용자 (CSO 운영) | 외부 시스템 / 자동화 워크플로우 |
| 인증 헤더 | `Authorization: Bearer ${COCSO_CLIENT_KEY}` | `Authorization: Bearer ${COCSO_SERVICE_KEY}` |
| 환경 변수 | `COCSO_CLIENT_MCP_URL` + `COCSO_CLIENT_KEY` | `COCSO_SERVICE_MCP_URL` + `COCSO_SERVICE_KEY` |

비즈니스사 사용자가 일상에서 쓰는 건 대부분 **Client MCP**. Service MCP는 외부 시스템 연동·자동화 시나리오용이라 직접 호출은 드뭅니다.

### 자동 등록 동작

COCO는 `.env` 또는 환경에 위 변수 한 쌍이 있으면 별도 `config.yaml` 편집 없이 자동으로 해당 MCP 서버를 등록합니다.

| URL | KEY | 결과 |
|---|---|---|
| 있음 | 있음 | 정상 등록 + 인증 헤더 부착 |
| 있음 | 없음 | URL만 등록 (anonymous — 서버 측 거부 가능) |
| 없음 | 있음 | 등록 안 됨 (URL 필요) |
| 없음 | 없음 | 등록 안 됨 |

각 MCP는 **독립적으로** 설정 가능 — Client만 설정하고 Service 비워둬도 OK, 반대도 OK.

### COCO가 MCP를 쓰는 방식

1. 사용자 요청 분석 → 어느 MCP의 어느 tool을 부를지 판단
2. tool 호출 전 한 줄로 알림: "Client MCP에서 ○○ 조회합니다."
3. 결과 받아 **요약 + 근거 위치** 형태로 응답
4. 권한 밖 요청 (다른 회사 데이터, Admin 영역 등) 은 거절

> Admin / Settler / Webhook 같은 다른 서버 영역은 별도 MCP가 노출되지 않습니다. COCO는 Client + Service 두 MCP의 도구만 사용합니다.

### 셋업 / 진단

| 명령 | 용도 |
|---|---|
| `cocso setup` | 회사명 + 두 MCP URL/KEY 인터랙티브 입력 |
| `cocso mcp list` | 현재 등록된 MCP 서버 + 인증 헤더 상태 확인 |
| `cocso doctor` | "◆ COCSO MCP" 섹션에서 두 MCP 등록 / 인증 상태 자동 검증 |

설정 변경 시 gateway 자동 재시작 — 사용자가 수동으로 재시작할 필요 없음.

## 응답 가이드

사용자가 회사·플랫폼에 대해 물을 때:

| 질문 | 우선 답할 영역 |
|---|---|
| "COCSO가 뭐야?" | 한 문장 → 누가 쓰는지 |
| "뭘 디지털화해?" | "무엇을 디지털화합니까" 표 |
| "Client / Contract가 뭐야?" | 플랫폼 구조 > 서버 표 |
| "수수료 어떻게 계산돼?" | 워커 > Commission |
| "계약은 어떻게 진행돼?" | 계약 워커 섹션 |
| "MCP가 뭐야?" / "어떤 MCP 있어?" | "COCSO MCP" 섹션 두 MCP 비교 표 |
| "MCP 설정 어떻게?" | "COCSO MCP > 셋업 / 진단" 표 |

전체를 한 번에 쏟지 말고, 사용자가 묻는 영역만 짧게 답합니다. 더 알고 싶어 하면 그때 다음 섹션을 펼칩니다.
